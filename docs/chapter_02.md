### 1. IP 协议概述

IP 协议是 TCP/IP 协议的核心，也是 socket 网络编程的基础之一。
- **功能**：出现在每个 IP 数据报中，用于指定 IP 通信的源端 IP 地址、目的端 IP 地址，指导 IP 分片和重组，以及指定部分通信行为。
- **发生位置**：除目标机器之外的所有主机和路由器上。
- **功能**：决定数据报是否应该转发以及如何转发。

---

## 2. IP 协议的特性

IP 协议为上层协议提供无状态、无连接、不可靠的服务。
- IP 通信双方不同步传输数据的状态信息，所有 IP 数据报的发送、传输和接收都是相互独立、没有上下文关系的。。
- IP 通信双方都不长久地维持对方的任何信息。
- IP 协议不能保证 IP 数据报准确地到达接收端，它只是承诺尽最大努力。

---

### 3. IP 分片

#### 分片机制
- **触发条件**：当 IP 数据报的长度超过帧的 MTU 时，它将被分片传输。
- **分片位置**：分片可能发生在发送端，也可能发生在中转路由器上，且可能在传输过程中被多次分片。
- **重组位置**：只有在最终的目标机器上，这些分片才会被内核中的 IP 模块重新组装。

#### IP 头部字段
- **数据报标识**：用于标识同一个 IP 数据报的所有分片。
- **标志**：MF（More Fragments）标志表示是否还有后续分片。
- **片偏移**：表示当前分片在原始数据报中的位置。
- **总长度字段**：每个分片的 IP 头部的总长度字段将被设置为该分片的长度。

#### IP 分组
- **定义**：IP 层传递给数据链路层的数据可能是一个完整的 IP 数据报，也可能是一个 IP 分片，它们统称为 IP 分组（packet）。

---

### 4. IP 路由

#### 路由机制
- **步骤 1**：查找路由表中和数据报的目标 IP 地址完全匹配的主机 IP 地址。如果找到，就使用该路由项；没找到则转步骤 2。
- **步骤 2**：查找路由表中和数据报的目标 IP 地址具有相同网络 ID 的网络 IP 地址。如果找到，就使用该路由项；没找到则转步骤 3。
- **步骤 3**：选择默认路由项，这通常意味着数据报的下一跳路由是网关。

#### 路由表更新
- **静态更新**：通过 `route` 命令或其他工具手工修改路由表。
- **动态更新**：大型路由器通常通过 BGP、RIP、OSPF 等协议来发现路径，并更新自己的路由表。
- **ICMP 重定向报文**：用于更新路由表。主机只能接收 ICMP 重定向报文，而路由器只能发送 ICMP 重定向报文。

---

### 5. IP 转发

#### 数据报转发
- **默认行为**：主机一般只发送和接收数据报，因为 `/proc/sys/net/ipv4/ip_forward` 内核参数默认被设置为 0。
- **启用转发**：可以通过以下命令启用主机的数据报转发功能：
```bash
  echo 1 > /proc/sys/net/ipv4/ip_forward
```
